    Introduction

    Welcome to the Food-API reference

    Users

    Register with username

    Make a POST request to /api/v1/user/create :

    curl -X POST --data "username=user_name" "http://someserver.com/api/v1/user/create"

    Make sure to replace user_name with the desired value

    The above command (if successful) returns JSON structured like this:

    {
  "id": 2,
  "name": "Pulkit Sharma",
  "api_key": "8413a762afed43fcf0c4983ac6033964",
  "created_at": "2016-02-11T06:59:59.268Z",
  "updated_at": "2016-02-11T06:59:59.268Z"
}

    If the above command is unsuccessful:

    {
  "status": "failure"
}

    Parameter Description

    Food Items

    List Food Items

    Make a GET request to /api/v1/food_items/view :

    curl -X GET --data "api_key=API_KEY&food_item_id=FOOD_ITEM_ID" "http://someserver.com/api/v1/food_items/view"

    Make sure to replace API_KEY,FOOD_ITEM_ID with the actual token value FOOD_ITEM_ID is an optional parameter.

    The above command (if successful) returns JSON structured like this:

    {
  "food_items": [
    {
      "id": 1,
      "name": "Pizza",
      "quantity_left": 97,
      "price": 60
    },
    {
      "id": 2,
      "name": "Burger",
      "quantity_left": 43,
      "price": 30
    }
  ]
}

    GET /api/v1/food_items/view

    If the `food_item_id` field is not provided, then all the food_items will be returned. If it is present, then only the concerned food_item is returned

    Query Parameters

    Parameter Description
    api_key The api_key provided to the user
    food_item_id (optional) The id of some food_item

    Create Food Items

    Make a POST request to /api/v1/food_items/create :

    curl -X POST --data "api_key=API_KEY&name=FOOD_NAME&price=PRICE&quantity=QTY" "http://someserver.com/api/v1/food_items/create"

    Make sure to replace API_KEY,FOOD_NAME,PRICE,QTY with the actual token value

    The above command (if successful) returns JSON structured like this:

    {
  "food_item": {
    "id": 3,
    "name": "Maggi",
    "quantity_left": 10,
    "price": 15
  }
}

    If the above command is unsuccessful:

    api_key The api_key provided to the user

    Orders

    Placing Orders

    Make a POST request to /api/v1/order/create :

    curl -X POST --data "api_key=API_KEY&items[][food_item_id]=FOOD_ITEM_ID_1&items[][quantity]=QTY_1" "http://someserver.com/api/v1/order/create"

    Make sure to replace API_KEY,FOOD_ITEM_ID_1,QTY_1 with the actual token value The above command adds only a single item , more items can be added by providing additional fields in a similar manner

    The above command (if successful) returns JSON structured like this:

    {
  "order": {
    "id": 15,
    "delivered": false,
    "payment": "pending",
    "total_price": "300",
    "order_details": [
      {
        "id": 17,
        "quantity": 3,
        "item": "Pizza",
        "price": "60"
      },
      {
        "id": 18,
        "quantity": 4,
        "item": "Burger",
        "price": "30"
      }
    ]
  }
}

    If the above command is unsuccessful:

    {
  "status": "failure"
}

    POST /api/v1/order/create

    This endpoint can be used to create an order containing multiple food_item with respective quantities

    Query Parameters

    Parameter Description
    api_key The api_key provided to the user
    items[][food_item_id] Id of the food_item (1)
    items[][quantity] Quantity of the food_item (1) placed in the order
    items[][food_item_id] (optional) Id of the food_item (2)
    items[][quantity] (optional) Quantity of the food_item (2) placed in the order

    More items can be added in a similar manner

    Updating a Placed Order

    Make a PUT request to /api/v1/order/update :

    curl -X PUT --data "api_key=API_KEY&order_id=ORDER_ID&order_detail_id=ORDER_DETAIL_ID&food_item_id=FOOD_ITEM_ID&quantity=QTY" "http://someserver.com/api/v1/order/update"

    Make sure to replace API_KEY,ORDER_ID,ORDER_DETAIL_ID, FOOD_ITEM_ID , QTY with the actual token value

    The above command (if successful) returns JSON structured like this:

    {
  "order": {
    "id": 18,
    "delivered": false,
    "payment": "pending",
    "total_price": "960",
    "order_details": [
      {
        "id": 23,
        "quantity": 4,
        "item": "Maggi",
        "price": "15"
      },
      {
        "id": 24,
        "quantity": 30,
        "item": "Burger",
        "price": "30"
      }
    ]
  }
}

    If the above command is unsuccessful:

    {
  "status": "failure"
}

    PUT /api/v1/order/update

    Query Parameters

    Parameter Description
    api_key The api_key provided to the user
    order_id Id of the order to be updated
    order_detail_id Id of the order_detail element present in the order , which needs to be updated
    food_item_id Id of the updated food_item
    quantity Quantity required of the food_item
